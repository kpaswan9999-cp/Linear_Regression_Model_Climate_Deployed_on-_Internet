End-to-End Workflow

📊

Step 1 — Data Collection & Preprocessing

Historical climate data is loaded from a structured CSV dataset. Missing values are filled with column medians and all inputs are converted to numeric types to ensure model compatibility.

🔬

Step 2 — Feature Selection

Eight meteorological features are selected as model inputs: Temperature Max, Temperature Min, Humidity, Rainfall, Wind Speed, AQI, Pressure, and Cloud Cover. The target variable is Average Temperature (°C).

🧠

Step 3 — Model Training

The dataset is split 80/20 for training and testing. A Linear Regression model is trained on the training split using scikit-learn. The resulting model is serialized to model.pkl for fast, stateless prediction.

✅

Step 4 — Model Validation

Both training and test R² scores are computed and stored in model_assessment.txt. If the gap between them exceeds 5%, an overfitting warning is logged. The current model shows a stable, near-perfect R² on both splits.

🚀

Step 5 — Live Prediction via Flask API

Users submit 8 meteorological values through the web interface. The Flask server loads the pre-trained model, builds a DataFrame from the inputs, and returns a real-time temperature prediction alongside model accuracy metrics and a feature visualization chart.
